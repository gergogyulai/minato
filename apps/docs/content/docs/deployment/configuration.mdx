## Environment variables

| Variable | Required | Description |
|---|---|---|
| `BETTER_AUTH_SECRET` | ✅ | Secret key used to sign sessions and tokens. Minimum 32 characters. Generate with `openssl rand -base64 32`. |
| `MEILISEARCH_MASTER_KEY` | ✅ | Meilisearch master key. Generate with `openssl rand -hex 32`. Must match `MEILI_MASTER_KEY` on the Meilisearch container. |
| `TMDB_READ_ACCESS_TOKEN` | ✅ | TMDB v4 Read Access Token for movie and TV metadata enrichment. Get one free at [themoviedb.org](https://developer.themoviedb.org/docs/getting-started). |
| `DATABASE_URL` | ✅ | PostgreSQL connection string, e.g. `postgresql://user:password@postgres:5432/minato`. |
| `REDIS_URL` | ✅ | Redis connection string. Defaults to `redis://localhost:6379` if not set. |
| `MEILISEARCH_HOST` | ✅ | Meilisearch base URL, e.g. `http://meilisearch:7700`. |
| `MEDIA_ROOT` | ✅ | Path inside the container where poster and backdrop images are stored. Use `/data/media`. |
| `BETTER_AUTH_URL` | | Public URL of your deployment, e.g. `https://minato.example.com`. Required when accessing Minato from a domain other than localhost. |
| `CORS_ORIGIN` | | Allowed CORS origin. Set this to your public URL when the frontend is accessed over a domain. |
| `NODE_ENV` | | Set to `production`. The Docker image sets this by default. |

### Generating secrets

```bash
# BETTER_AUTH_SECRET
openssl rand -base64 32

# MEILISEARCH_MASTER_KEY
openssl rand -hex 32
```

Don't use passwords you've used elsewhere. These values are security-sensitive — if either leaks, rotate them and restart the stack.

### Rotating a secret

To rotate `BETTER_AUTH_SECRET`, update it in your `.env` file and restart the stack. Existing user sessions will be invalidated and users will need to log in again.

To rotate `MEILISEARCH_MASTER_KEY`, update it in your `.env` file, update `MEILI_MASTER_KEY` on the Meilisearch container, and restart the stack.

## Volumes

All state lives in Docker named volumes. Back these up to avoid data loss.

| Volume | What it holds |
|---|---|
| `minato_assets` | Poster and backdrop images, stored under `/data/media` inside the container |
| `postgres_data` | The PostgreSQL data directory — your entire torrent database |
| `redis_data` | Redis persistence for the BullMQ job queue |
| `meilisearch_data` | Meilisearch index data |

The Meilisearch index can be rebuilt from PostgreSQL if lost — it's not the source of truth. The other three cannot be reconstructed, so treat them as critical data.

### Backing up

```bash
# Postgres — take a logical dump
docker exec minato-postgres pg_dump -U minato_user minato > minato_$(date +%Y%m%d).sql

# Full volume backup (works for any volume)
docker run --rm \
  -v minato_assets:/data \
  -v $(pwd):/backup \
  alpine tar czf /backup/minato_assets_$(date +%Y%m%d).tar.gz -C /data .
```

### Restoring Postgres

```bash
cat minato_backup.sql | docker exec -i minato-postgres psql -U minato_user minato
```

## Ports

Only one port needs to be exposed to the host.

| Port | Service |
|---|---|
| `7271` | nginx — serves the web frontend and proxies `/api/` requests to the Hono backend |

All other services (PostgreSQL, Redis, Meilisearch) communicate on the internal Docker network and are not exposed. Don't expose them unless you have a specific reason to.

## Community scraper volume

If you plan to use community scrapers, mount a host directory into the container at `/app/scrapers`:

```yaml
services:
  minato:
    volumes:
      - minato_assets:/data
      - ./scrapers:/app/scrapers   # add this line
```

Drop scraper directories into `./scrapers/` on your host. Minato picks them up automatically without a restart.
